# Documentation Writing Skills

## KMP Documentation

- Understand Kotlin Multiplatform structure
- Document shared business logic
- Explain platform-specific implementations
- Include build configurations and commands

## Clean Architecture Documentation

- Document domain layer (use cases, interfaces)
- Explain data layer implementations
- Describe presentation layer patterns
- Show data flow between layers

## Technical Documentation Standards

- Use clear, concise language
- Include code examples
- Provide step-by-step instructions
- Maintain consistency across documents

## Documentation Structure

- Start with overview/summary
- Provide detailed sections
- Include quick reference guides
- End with practical examples

## Code Documentation

- Document public APIs
- Explain architecture decisions
- Include usage examples
- Maintain inline comments

## Command Documentation

- Provide exact command syntax
- Include platform-specific variations
- Explain command purposes
- Add troubleshooting tips
